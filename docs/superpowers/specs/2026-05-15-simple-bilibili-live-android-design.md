# simpleBilibiliLive Android 应用设计规格

## 背景

目标是在小米小爱触屏音箱上安装一款轻量安卓应用，用于观看哔哩哔哩直播。设备按安卓 8 兼容、800x400 分辨率、默认横屏使用设计。首版以性能和稳定性优先，不追求完整 B 站客户端能力。

## 已确认范围

- 首版只支持指定直播间观看，可通过输入房间号或最近记录进入。
- 支持扫码登录自己的 B 站账号，并持久化登录态。
- 允许使用 B 站非公开接口获取二维码登录状态、房间信息和直播流地址。
- 播放链路优先使用原生播放器，不用 WebView 承担核心播放。
- 首版不显示弹幕。
- 首版不做推荐页、分区浏览、搜索、历史动态、完整关注管理等重型功能。

## 技术路线

采用原生 Android 应用：

- 语言：Kotlin。
- 最低系统：Android 8.0，`minSdk 26`。
- 播放器：AndroidX Media3 ExoPlayer。
- 网络：OkHttp。
- JSON：kotlinx.serialization 或 Moshi，实施时根据工程模板和依赖兼容性确认。
- 持久化：DataStore 或 SharedPreferences；首版数据量小，优先选择稳定、简单的方案。
- UI：横屏原生界面，可使用 View/XML 或 Compose。若目标设备对 Compose 运行成本敏感，优先 View/XML。

## 设备与交互约束

- 应用强制横屏，按 800x400 首屏布局设计。
- 播放页默认全屏，减少层级、动画和透明叠加。
- 文本和按钮必须适合近距离触控，主操作区域不依赖细小控件。
- 所有页面避免整体滚动，关键操作在一屏内完成。
- 默认进入上次使用的房间，减少设备端输入成本。
- 播放中只保留必要控件：返回、刷新、清晰度、音量/亮度入口、错误重试。

## 核心模块

### auth

职责：

- 请求二维码登录凭据。
- 展示二维码。
- 轮询扫码状态。
- 保存登录 Cookie/token。
- 检查登录态是否有效。
- 登录失效时清理本地状态并回到登录页。

边界：

- 不在首版实现账号资料页。
- 不在首版实现多账号切换。

### bili-api

职责：

- 根据房间号获取真实房间信息。
- 判断直播间状态：直播中、未开播、房间不存在、被限制访问。
- 获取可用播放地址和清晰度列表。
- 将接口返回统一转换为应用内部模型。

边界：

- 非公开接口集中封装在该模块，避免散落到 UI 或播放器代码中。
- 接口变化时优先只修改该模块。

### player

职责：

- 使用 Media3 ExoPlayer 播放直播流。
- 支持清晰度切换。
- 支持播放失败重试。
- 支持网络抖动后的自动恢复。
- 处理应用前后台、音频焦点和播放器释放。

策略：

- 默认选择兼顾稳定和性能的清晰度，不强制最高画质。
- 首次失败快速重试，连续失败后显示错误覆盖层。
- 不渲染弹幕，降低 CPU/GPU 压力。

### storage

职责：

- 保存登录态。
- 保存最近房间号。
- 保存清晰度偏好。
- 保存基础设置。

### ui

页面状态：

- 启动检查：加载登录态、网络状态、最近房间。
- 登录页：显示二维码和扫码状态。
- 房间页：输入房间号，展示最近房间。
- 播放页：横屏全屏播放和轻量控制层。
- 错误覆盖层：登录失效、取流失败、未开播、网络失败。

## 数据流

启动流程：

1. App 启动并锁定横屏。
2. 读取本地登录态和最近房间。
3. 若没有登录态，进入扫码登录。
4. 若有登录态且有最近房间，尝试进入该房间播放。
5. 若无最近房间，进入房间输入页。

扫码登录流程：

1. `auth` 请求二维码凭据。
2. UI 根据二维码内容生成图片。
3. `auth` 轮询扫码状态。
4. 登录成功后保存登录态。
5. 进入房间输入页或恢复上次房间。

播放流程：

1. UI 提交房间号。
2. `bili-api` 获取房间信息和直播状态。
3. 若直播中，获取播放地址列表。
4. `player` 选择默认播放地址并开始播放。
5. 播放失败时由 `player` 自动重试，超过阈值后通知 UI 显示错误覆盖层。

## 错误处理

- 二维码过期：自动刷新二维码，并提示重新扫码。
- 登录失效：清理登录态，回到登录页。
- 房间不存在：停留在房间页并提示检查房间号。
- 未开播：显示未开播状态，允许手动刷新。
- 取流失败：尝试降级清晰度或切换备用地址。
- 网络失败：播放器短暂自动重试；连续失败后显示重试按钮。
- 接口返回格式变化：记录原始错误摘要，提示“接口异常，需更新适配”。

## 性能原则

- 播放页不使用 WebView。
- 默认关闭弹幕和高频动画。
- UI 控制层只在触摸或按键操作时短暂显示。
- 图片资源和背景尽量少，避免重型视觉效果。
- 网络请求设置明确超时和重试策略。
- 播放器生命周期严格释放，避免长时间使用后内存增长。
- 首版不引入大体积跨端运行时。

## 测试与验收

最低验收项：

- 安卓 8 模拟器或真机可以安装启动。
- 应用默认横屏显示，800x400 下无关键控件溢出。
- 可以显示二维码并完成扫码登录。
- 可以输入房间号并进入直播播放。
- 未开播、房间不存在、登录失效均有明确提示。
- 直播播放 30 分钟无崩溃、无明显内存持续增长。
- 断网后显示错误状态，恢复网络后可手动重试播放。

## 非目标

- 不实现弹幕。
- 不实现直播推荐流。
- 不实现搜索。
- 不实现礼物、互动、发送弹幕。
- 不实现完整个人中心。
- 不保证非公开接口长期不变，接口变更时通过 `bili-api` 模块维护。
